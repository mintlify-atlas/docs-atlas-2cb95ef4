## System Architecture

SAFI ControlHub is a full-stack web application designed for real-time monitoring and control of aerospace test systems. The architecture consists of three main layers:

```mermaid
graph TB
    subgraph Hardware Layer
        Arduino[Arduino/ESP32 Devices]
        Serial[Serial Ports]
    end
    
    subgraph Backend Layer
        Python[Python Scripts]
        Express[Express Server]
        WS[WebSocket Server]
        DB[(PostgreSQL)]
    end
    
    subgraph Frontend Layer
        React[React Application]
        Context[Context Providers]
        Components[UI Components]
    end
    
    Arduino -->|Serial Communication| Python
    Python -->|HTTP POST| Express
    Express -->|SQL Queries| DB
    Express -->|Real-time Updates| WS
    WS <-->|Socket.IO| React
    React -->|HTTP/REST| Express
    Context --> Components
```

## Technology Stack

### Backend

- **Runtime**: Node.js with ES Modules
- **Framework**: Express.js
- **Database**: PostgreSQL with `pg` pool
- **Real-time**: Socket.IO for WebSocket communication
- **Authentication**: JWT (JSON Web Tokens) with cookie-based sessions
- **Hardware Integration**: Python scripts for serial communication

### Frontend

- **Framework**: React 18 with Vite
- **Routing**: React Router v6
- **State Management**: React Context API
- **UI Library**: Material-UI (MUI) with custom theme
- **Real-time**: Socket.IO client
- **HTTP Client**: Axios
- **3D Visualization**: React Three Fiber (@react-three/fiber)

### Hardware Communication

- **Serial Library**: pySerial (Python) and serialport (Node.js)
- **Protocols**: Serial UART communication over USB
- **Supported Devices**: Arduino, ESP32

## Core Components

### Server Entry Point

The application starts in `src/index.js:1`:

```javascript
import app from "./app.js";
import { createServer } from "http";
import { initWebSocket } from "./websocket.js";

const server = createServer(app);
initWebSocket(server);
server.listen(PORT);
```

This creates an HTTP server that serves both the REST API and WebSocket connections.

### Database Connection

PostgreSQL connection pool configured in `src/db.js:1`:

```javascript
export const pool = new pg.Pool({
  port: PG_PORT,
  host: PG_HOST,
  user: PG_USER,
  password: PG_PASSWORD,
  database: PG_DATABASE,
});
```

The pool is used throughout the application for executing SQL queries.

## Data Flow Architecture

### 1. Hardware to Database Flow

```mermaid
sequenceDiagram
    participant Arduino
    participant Python
    participant Express
    participant Database
    participant WebSocket
    participant Client
    
    Arduino->>Python: Serial Data (9600/115200 baud)
    Python->>Python: Parse & Format JSON
    Python->>Express: POST /api/bancodepruebas
    Express->>Database: INSERT query
    Database-->>Express: Success
    WebSocket->>Database: Poll for changes
    Database-->>WebSocket: New data
    WebSocket->>Client: Emit event
```

### 2. Client to Server Flow

```mermaid
sequenceDiagram
    participant Client
    participant Express
    participant Middleware
    participant Controller
    participant Database
    
    Client->>Express: HTTP Request
    Express->>Middleware: CORS Check
    Middleware->>Middleware: JWT Validation
    Middleware->>Controller: Route to handler
    Controller->>Database: Execute query
    Database-->>Controller: Result
    Controller-->>Client: JSON Response
```

## Authentication Flow

JWT-based authentication with HTTP-only cookies:

```mermaid
sequenceDiagram
    participant Client
    participant Login
    participant Auth
    participant Database
    
    Client->>Login: POST /api/login
    Login->>Database: Verify credentials
    Database-->>Login: User data
    Login->>Login: Sign JWT token
    Login-->>Client: Set cookie + user data
    Client->>Auth: Request with cookie
    Auth->>Auth: Verify JWT (isAuth middleware)
    Auth-->>Client: Authorized response
```

The `isAuth` middleware (`src/middlewares/auth.middleware.js:3`) validates tokens on protected routes.

## Environment Configuration

The backend uses environment variables for configuration (`src/config.js:1`):

- **PORT**: Server port (default: 3000)
- **ORIGIN**: CORS allowed origins
- **PG_HOST, PG_PORT, PG_USER, PG_PASSWORD, PG_DATABASE**: PostgreSQL connection

Multiple development origins are whitelisted in `src/app.js:17-25` for local network access.
