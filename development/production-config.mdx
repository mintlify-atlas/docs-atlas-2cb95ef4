## Database Optimization

### Connection Pool Configuration

Update `src/db.js:10-16` to optimize PostgreSQL connection pooling:

```javascript
// src/db.js
import pg from "pg";
import {
  PG_DATABASE,
  PG_HOST,
  PG_PASSWORD,
  PG_PORT,
  PG_USER,
} from "./config.js";

export const pool = new pg.Pool({
  port: PG_PORT,
  host: PG_HOST,
  user: PG_USER,
  password: PG_PASSWORD,
  database: PG_DATABASE,
  // Production optimizations:
  max: 20,                          // Maximum pool size
  idleTimeoutMillis: 30000,         // Close idle clients after 30s
  connectionTimeoutMillis: 2000,    // Return error if connection takes > 2s
  maxUses: 7500,                    // Close connections after 7500 uses
  allowExitOnIdle: false,           // Keep process alive with idle pool
});

pool.on("connect", () => {
  console.log("Database connected");
});

pool.on("error", (err) => {
  console.error("Unexpected database error:", err);
});
```

| Query | Interval | Table | Optimization |
|-------|----------|-------|-------------|
| `nuevos-datos` | 100ms | `data` | Add index on `(date DESC, time DESC)` |
| `banco-datos` | 500ms | `prueba_estatica_0` | Add index on `id DESC` |
| `xitzin2-datos` | 200ms | `xitzin_2_data` | Consider materialized view |
| `baterias` | 5000ms | `battery_status` | Add composite index |
| `xitzin2-baterias` | 5000ms | `xitzin_2_batteries` | Add composite index |
| `laniakea` | 200ms | `datos_laniakea` | Add index on `(date DESC, time DESC)` |

#### Add Database Indexes

```sql
-- Optimize frequent DESC queries (websocket.js:77,83,107)
CREATE INDEX idx_data_date_time ON data (date DESC, time DESC);
CREATE INDEX idx_prueba_estatica_id ON prueba_estatica_0 (id DESC);
CREATE INDEX idx_laniakea_date_time ON datos_laniakea (date DESC, time DESC);
```

### Connection Handling

Add connection limits and rate limiting:

```javascript
// src/websocket.js
export function initWebSocket(server) {
  io = new Server(server, {
    cors: { /* ... */ },
    maxHttpBufferSize: 1e6,              // 1MB max message size
    pingTimeout: 60000,                  // 60s ping timeout
    pingInterval: 25000,                 // Ping every 25s
    transports: ['websocket', 'polling'], // Prefer WebSocket
    allowUpgrades: true,
  });

  // Track connections per IP
  const connectionCount = new Map();

  io.use((socket, next) => {
    const ip = socket.handshake.address;
    const count = connectionCount.get(ip) || 0;

    if (count > 10) {
      return next(new Error('Too many connections from this IP'));
    }

    connectionCount.set(ip, count + 1);
    socket.on('disconnect', () => {
      connectionCount.set(ip, (connectionCount.get(ip) || 1) - 1);
    });

    next();
  });
  
  // ... rest of implementation
}
```

<Warning>
  The current implementation emits to **all connected clients** on every data change. With many clients, this can cause significant server load. Consider implementing rooms or namespaces for different data channels.
</Warning>

```javascript
// src/websocket.js - cache high-frequency queries
import { redis } from './cache.js';

const emitIfNew = async ({ query, channel, getLast, cacheKey, ttl }) => {
  // Try cache first
  const cached = await redis.get(cacheKey);
  if (cached) {
    const rows = JSON.parse(cached);
    io.emit(channel, rows);
    return;
  }
  
  // Query database and cache
  const { rows } = await pool.query(query);
  if (rows.length > 0) {
    await redis.setEx(cacheKey, ttl, JSON.stringify(rows));
    io.emit(channel, rows);
  }
};
```

## Troubleshooting

### High Database Load

**Symptom**: PostgreSQL CPU > 80%

**Cause**: WebSocket polling queries (websocket.js:76-110)

**Solution**:
1. Add database indexes (see Query Optimization section)
2. Implement Redis caching for high-frequency queries
3. Increase polling intervals for non-critical data
4. Use `EXPLAIN ANALYZE` to identify slow queries

### WebSocket Disconnections

**Symptom**: Clients frequently disconnect/reconnect

**Cause**: Aggressive ping/pong timeouts or reverse proxy configuration

**Solution**:
```javascript
// Increase timeout values
io = new Server(server, {
  pingTimeout: 60000,  // Increase from default 20000
  pingInterval: 25000, // Increase from default 25000
});
```

Update Nginx:
```nginx
proxy_read_timeout 90s;
proxy_send_timeout 90s;
```

### CORS Errors

**Symptom**: Browser shows CORS policy errors

**Cause**: Frontend domain not in allowedOrigins (app.js:17-25)

**Solution**: Add your production domain to the allowedOrigins array and restart the backend

### Memory Leaks

**Symptom**: Node.js process memory grows over time

**Cause**: Possible issues in WebSocket event handlers or database connection leaks

**Solution**:
1. Monitor with `pm2 monit`
2. Use `--max-old-space-size=2048` to increase heap limit
3. Review WebSocket disconnect handlers
4. Ensure database connections are properly released
