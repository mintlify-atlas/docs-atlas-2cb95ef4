## Overview

This guide will walk you through setting up SAFI ControlHub and monitoring your first device telemetry data. You'll have a working installation with live data streams in just a few minutes.

## Prerequisites

Before you begin, make sure you have:

- **Node.js** (v16 or higher)
- **PostgreSQL** (v12 or higher)
- A serial-connected device (optional, for live data)

<Steps>

### Clone the Repository

First, get the source code:

```bash
git clone <repository-url>
cd source
```

### Set Up the Database

Create a PostgreSQL database and initialize the schema:

```bash
psql -U postgres
```

Then in the PostgreSQL shell:

```sql
CREATE DATABASE rocket_data;
\c rocket_data
\i database/init.sql
```

<Note>
The default database name is `rocket_data`. You can change this in your environment configuration.
</Note>

### Configure Environment Variables

Create a `.env` file in the root directory with your database credentials:

```bash
PORT=3000
PG_HOST=localhost
PG_PORT=5432
PG_USER=postgres
PG_PASSWORD=admin@123
PG_DATABASE=rocket_data
ORIGIN=http://localhost:5173
```

<Warning>
Make sure to change `PG_PASSWORD` from the default value for production deployments.
</Warning>

### Install Backend Dependencies

Install all required packages:

```bash
npm install
```

Key dependencies include:
- `express` - Web server framework
- `socket.io` - Real-time WebSocket communication
- `pg` - PostgreSQL client
- `serialport` - Serial device communication

### Start the Backend Server

Run the development server:

```bash
npm run dev
```

You should see:

```
Database connected
Servidor HTTP + WS corriendo en puerto 3000
```

Test the connection:

```bash
curl http://localhost:3000/api/ping
```

### Install Frontend Dependencies

In a new terminal, navigate to the frontend directory:

```bash
cd frontend
npm install
```

### Start the Frontend

Launch the React development server:

```bash
npm run dev
```

The frontend will be available at `http://localhost:5173`

### Access the Dashboard

Open your browser and navigate to:

```
http://localhost:5173
```

You should see the SAFI ControlHub dashboard.

</Steps>

## Verify Live Telemetry

Once both servers are running, the WebSocket connection will automatically:

1. **Connect** to the backend on port 3000
2. **Load initial data** from all telemetry tables
3. **Stream updates** in real-time:
   - Main telemetry data (100ms intervals)
   - Battery status (5s intervals)
   - Test bench data (500ms intervals)
   - Laniakea mission data (200ms intervals)

<CodeGroup>

```javascript WebSocket Channels
// Channels emitting real-time data:
"nuevos-datos"        // Main telemetry (AKBAL-II)
"banco-datos"         // Test bench data
"xitzin2-datos"       // XITZIN-II telemetry
"baterias"            // Battery status
"xitzin2-baterias"    // XITZIN-II batteries
"laniakea"            // Laniakea mission data
"ignicion-estado"     // Ignition commands
```

```bash Test API Endpoints
# Check database connection
curl http://localhost:3000/api/ping

# Get telemetry data
curl http://localhost:3000/api/data

# Get battery status
curl http://localhost:3000/api/batteries

# Get Laniakea data
curl http://localhost:3000/api/laniakea
```

</CodeGroup>

## Send Test Data

To verify the full pipeline, send test telemetry:

```bash
curl -X POST http://localhost:3000/api/data \
  -H "Content-Type: application/json" \
  -d '{"temperature": 25.5, "pressure": 101.3}'
```

You should see the data appear in the dashboard immediately via WebSocket.

## Troubleshooting

### Database Connection Failed

If you see `Database connected` not appearing:

1. Verify PostgreSQL is running: `pg_isready`
2. Check credentials in `.env` file
3. Ensure database exists: `psql -l | grep rocket_data`

### WebSocket Not Connecting

If the frontend can't connect:

1. Verify backend is running on port 3000
2. Check CORS configuration in `src/app.js:17`
3. Ensure your origin is in the `allowedOrigins` array

### Port Already in Use

If port 3000 or 5173 is occupied:

```bash
# Backend
PORT=3001 npm run dev

# Frontend
vite --port 5174
```

Update the `ORIGIN` environment variable accordingly.
