## Overview

SAFI ControlHub broadcasts real-time telemetry data through six primary WebSocket channels. Each channel has a specific update interval and only emits when data has changed, reducing unnecessary network traffic.

## nuevos-datos

<div className="mb-4">
  <Badge>100ms update interval</Badge>
  <Badge>150 records per emission</Badge>
</div>

Main telemetry channel for the Xitzin-II rocket system.

### Configuration

**Source:** `src/websocket.js:76-80`

```javascript
setInterval(() => emitIfNew({
  query: "SELECT * FROM data ORDER BY date DESC, time DESC LIMIT 150",
  channel: "nuevos-datos",
  getLast: (rows) => `${rows[0]?.date}-${rows[0]?.time}`,
}), 100);
```

### Data Fields

<ParamField body="date" type="string">
  Recording date (YYYY-MM-DD)
</ParamField>

<ParamField body="time" type="string">
  Recording time (HH:MM:SS)
</ParamField>

<ParamField body="altitude" type="number">
  Altitude in meters
</ParamField>

<ParamField body="temperature" type="number">
  Temperature in Celsius
</ParamField>

<ParamField body="pressure" type="number">
  Atmospheric pressure
</ParamField>

<ParamField body="velocity" type="number">
  Velocity in km/h
</ParamField>

<ParamField body="latitude" type="number">
  GPS latitude coordinate
</ParamField>

<ParamField body="longitude" type="number">
  GPS longitude coordinate
</ParamField>

<ParamField body="accel_x" type="number">
  X-axis acceleration (m/s²)
</ParamField>

<ParamField body="accel_y" type="number">
  Y-axis acceleration (m/s²)
</ParamField>

<ParamField body="accel_z" type="number">
  Z-axis acceleration (m/s²)
</ParamField>

<ParamField body="mission_state" type="integer">
  Mission phase:
  - `0` = Preflight
  - `1` = Lift-off
  - `2` = Apogee
  - `3` = Drogue
  - `4` = Main
  - `5` = Land
</ParamField>

<ParamField body="air_brake_angle" type="number">
  Air brake angle in degrees
</ParamField>

### Usage Example

```javascript
import useWebsocket from '../api/useWebsocket';
import { SOCKET_URL } from '../api/socketConfig';

function TelemetryDashboard() {
  const rawData = useWebsocket('nuevos-datos', SOCKET_URL);
  
  // Extract data arrays for visualization
  const altitude = rawData.map(d => parseFloat(d.altitude) || 0);
  const velocity = rawData.map(d => parseFloat(d.velocity) || 0);
  const temperature = rawData.map(d => parseFloat(d.temperature) || 0);
  
  return (
    <div>
      <h2>Live Telemetry</h2>
      <p>Records: {rawData.length}</p>
      <p>Latest altitude: {altitude[0]} m</p>
      <p>Latest velocity: {velocity[0]} km/h</p>
    </div>
  );
}
```

## banco-datos

<div className="mb-4">
  <Badge>500ms update interval</Badge>
  <Badge>100 records per emission</Badge>
</div>

Static test bench data for ground testing and validation.

### Configuration

**Source:** `src/websocket.js:82-86`

```javascript
setInterval(() => emitIfNew({
  query: "SELECT * FROM prueba_estatica_0 ORDER BY id DESC LIMIT 100",
  channel: "banco-datos",
  getLast: (rows) => rows[0]?.id,
}), 500);
```

### Change Detection

Updates are triggered by changes to the `id` field (auto-incrementing primary key).

### Data Fields

<ParamField body="id" type="integer">
  Unique record identifier
</ParamField>

<ParamField body="[additional fields]" type="various">
  Test bench specific measurements and configurations
</ParamField>

### Usage Example

```javascript
const testBenchData = useWebsocket('banco-datos', SOCKET_URL);

console.log(`Test bench records: ${testBenchData.length}`);
if (testBenchData.length > 0) {
  console.log(`Latest test ID: ${testBenchData[0].id}`);
}
```

## xitzin2-datos

<div className="mb-4">
  <Badge>200ms update interval</Badge>
  <Badge>All records</Badge>
</div>

Alternate telemetry stream for Xitzin-II system.

### Configuration

**Source:** `src/websocket.js:88-92`

```javascript
setInterval(() => emitIfNew({
  query: "SELECT * FROM xitzin_2_data",
  channel: "xitzin2-datos",
  getLast: (rows) => JSON.stringify(rows[0]),
}), 200);
```

<Info>
  Unlike other channels, this emits **all records** from the table, not a limited subset.
</Info>

### Change Detection

Updates are triggered by changes to the first record (serialized as JSON).

### Data Fields

Similar structure to `nuevos-datos` channel. Includes:
- Telemetry data (altitude, velocity, temperature, pressure)
- GPS coordinates (latitude, longitude)
- Acceleration vectors (accel_x, accel_y, accel_z)
- Mission state
- Timestamps (date, time)

### Usage Example

```javascript
const xitzin2Data = useWebsocket('xitzin2-datos', SOCKET_URL);

// Process all available data
const allAltitudes = xitzin2Data.map(d => parseFloat(d.altitude) || 0);
const maxAltitude = Math.max(...allAltitudes);
```

## baterias

<div className="mb-4">
  <Badge>5000ms update interval</Badge>
  <Badge>All battery records</Badge>
</div>

Battery monitoring for the main rocket system.

### Configuration

**Source:** `src/websocket.js:94-98`

```javascript
setInterval(() => emitIfNew({
  query: "SELECT * FROM battery_status",
  channel: "baterias",
  getLast: (rows) => JSON.stringify(rows),
}), 5000);
```

### Change Detection

Updates are triggered by changes to any battery record (all rows serialized as JSON).

### Data Fields

<ParamField body="battery_id" type="integer">
  Unique battery identifier
</ParamField>

<ParamField body="battery_level" type="number">
  Charge level percentage (0-100)
</ParamField>

<ParamField body="voltage" type="number">
  Current voltage in volts
</ParamField>

<ParamField body="temperature" type="number">
  Battery temperature in Celsius
</ParamField>

### Usage Example

```javascript
import { LinearProgress } from '@mui/material';

function BatteryStatus() {
  const batteryData = useWebsocket('baterias', SOCKET_URL);
  
  return (
    <div>
      {batteryData.map((battery) => (
        <div key={battery.battery_id}>
          <h3>Battery {battery.battery_id}</h3>
          <LinearProgress 
            variant="determinate" 
            value={battery.battery_level}
            color={battery.battery_level < 30 ? 'error' : 'success'}
          />
          <p>Level: {battery.battery_level}%</p>
          <p>Voltage: {battery.voltage} V</p>
          <p>Temp: {battery.temperature} °C</p>
        </div>
      ))}
    </div>
  );
}
```

## Performance Considerations

<AccordionGroup>
  <Accordion title="Intelligent Change Detection">
    The server uses timestamp-based change detection to avoid emitting duplicate data:
    
    **Source:** `src/websocket.js:64-74`
    
    ```javascript
    const emitIfNew = async ({ query, channel, getLast }) => {
      const { rows } = await pool.query(query);
      if (rows.length === 0) return;

      const newTimestamp = getLast(rows);
      if (newTimestamp !== lastTimestamps[channel]) {
        io.emit(channel, rows);
        lastTimestamps[channel] = newTimestamp;
      }
    };
    ```
    
    This ensures updates are only broadcast when data actually changes.
  </Accordion>

  <Accordion title="Optimized Update Intervals">
    Channels are configured with different intervals based on data volatility:
    
    - **High-frequency telemetry** (100-200ms): `nuevos-datos`, `xitzin2-datos`, `laniakea`
    - **Medium-frequency data** (500ms): `banco-datos`
    - **Low-frequency status** (5000ms): `baterias`, `xitzin2-baterias`
  </Accordion>

  <Accordion title="Record Limiting">
    Most channels limit the number of records to prevent excessive data transfer:
    
    - `nuevos-datos`: Last 150 records
    - `banco-datos`: Last 100 records
    - `laniakea`: Last 100 records
    - `xitzin2-datos`: All records (use cautiously)
  </Accordion>

  <Accordion title="Client-Side Recommendations">
    - Always validate and parse incoming data
    - Use the `useWebsocket` hook for automatic lifecycle management
    - Implement error boundaries for data display components
    - Consider throttling UI updates for high-frequency channels
  </Accordion>
</AccordionGroup>

## Multiple Channel Subscription

```javascript
import useWebsocket from '../api/useWebsocket';
import { SOCKET_URL } from '../api/socketConfig';

function MultiChannelDashboard() {
  // Subscribe to multiple channels simultaneously
  const xitzinData = useWebsocket('nuevos-datos', SOCKET_URL);
  const laniakeaData = useWebsocket('laniakea', SOCKET_URL);
  const batteryData = useWebsocket('baterias', SOCKET_URL);
  
  return (
    <div>
      <h2>Multi-System Dashboard</h2>
      
      <section>
        <h3>Xitzin-II</h3>
        <p>Records: {xitzinData.length}</p>
        {xitzinData.length > 0 && (
          <p>Altitude: {parseFloat(xitzinData[0].altitude) || 0} m</p>
        )}
      </section>
      
      <section>
        <h3>Laniakea</h3>
        <p>Records: {laniakeaData.length}</p>
        {laniakeaData.length > 0 && (
          <p>Altitude: {parseFloat(laniakeaData[0].altitud) || 0} m</p>
        )}
      </section>
      
      <section>
        <h3>Battery Status</h3>
        <p>Batteries monitored: {batteryData.length}</p>
      </section>
    </div>
  );
}
```
