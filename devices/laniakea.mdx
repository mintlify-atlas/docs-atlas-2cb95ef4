Laniakea is a standalone telemetry system providing GPS tracking, acceleration monitoring, and battery status for rocket flights.

## Overview

The Laniakea dashboard (`frontend/src/pages/Laniakea.jsx`) provides comprehensive telemetry monitoring:

- GPS position tracking with live map
- Multi-axis acceleration monitoring
- Voltage and current monitoring
- Battery status for up to 5 batteries
- Real-time D3.js charts

## Data Streams

### Primary Telemetry Channel: `laniakea`

**WebSocket Channel:** `laniakea`  
**Update Rate:** 200ms  
**Database Table:** `datos_laniakea`  
**Endpoint:** `GET/POST /api/laniakea`

<Accordion title="Telemetry Data Fields">

| Field | Type | Description | Units |
|-------|------|-------------|-------|
| `date` | string | Data timestamp date | - |
| `time` | string | Data timestamp time | - |
| `latitud` | float | GPS latitude | decimal degrees |
| `longitud` | float | GPS longitude | decimal degrees |
| `altitud` | float | GPS altitude | meters |
| `aceleracion` | float | Total acceleration magnitude | m/s² |
| `accel_x` | float | X-axis acceleration | m/s² |
| `accel_y` | float | Y-axis acceleration | m/s² |
| `accel_z` | float | Z-axis acceleration | m/s² |
| `conops` | string | Mission state identifier | - |
| `voltaje` | float | System voltage | V |
| `corriente` | float | System current | A |

</Accordion>

### Battery Monitoring Channel: `baterias`

**WebSocket Channel:** `baterias`  
**Update Rate:** 5 seconds  
**Database Table:** `battery_status`  
**Endpoint:** `GET/POST /api/batteries`

<Note>
Laniakea shares the same battery monitoring channel as AKBAL-II.
</Note>

## Hardware Communication

Laniakea uses an ESP32 microcontroller connected via USB serial.

### Python Data Bridge

**Script:** `src/arduino/laniakea.py`  
**Serial Port:** `/dev/ttyUSB0`  
**Baud Rate:** 115200

<Accordion title="Serial Data Format">

The ESP32 sends comma-separated values:

```
lat,long,alt
```

**Example:**
```
19.123456,-99.234567,2250.5
```

The Python script parses this and constructs a full JSON payload:

```python
payload = {
    "timestamp": datetime.now().isoformat(),
    "latitud": latitud,
    "longitud": longitud,
    "aceleracion": 0.0,
    "accel_x": 0.0,
    "accel_y": 0.0,
    "accel_z": 0.0,
    "altitud": altitud,
    "conops": "",
    "voltaje": 0.0,
    "corriente": 0.0
}
```

<Warning>
Acceleration, voltage, and current fields are currently hardcoded to 0.0 in the Python script. Only GPS data (lat, long, alt) is transmitted from the ESP32.
</Warning>

</Accordion>

### Data Flow

1. ESP32 reads GPS module and sends `lat,long,alt` via serial
2. Python script (`laniakea.py`) reads serial data
3. Data is posted to `POST /api/laniakea`
4. Backend stores in `datos_laniakea` table
5. WebSocket broadcasts on `laniakea` channel every 200ms
6. Frontend dashboard updates in real-time
