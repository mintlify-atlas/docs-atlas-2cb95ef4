The Filamentadora is a CNC-controlled composite winding system for manufacturing rocket motor casings with precise fiber placement.

## Overview

The Filamentadora system provides:

- **Automated Winding** - Computer-controlled mandrel rotation and carriage movement
- **Pattern Visualization** - Real-time 2D preview of winding patterns
- **Parameter Control** - Adjustable RPM, feed rate, angle, and layer count
- **Layer Management** - Multi-layer composite builds

## Control Commands

The system accepts the following commands via `POST /api/filamentadora`:

<CodeGroup>

```javascript START_ROT
// Start mandrel rotation
await axios.post('http://localhost:3000/api/filamentadora', {
  command: 'START_ROT'
});
```

```javascript STOP_ROT
// Stop mandrel rotation
await axios.post('http://localhost:3000/api/filamentadora', {
  command: 'STOP_ROT'
});
```

```javascript MOVE_HEAD_FWD
// Move carriage forward
await axios.post('http://localhost:3000/api/filamentadora', {
  command: 'MOVE_HEAD_FWD'
});
```

```javascript MOVE_HEAD_BWD
// Move carriage backward
await axios.post('http://localhost:3000/api/filamentadora', {
  command: 'MOVE_HEAD_BWD'
});
```

```javascript SET_RPM
// Set mandrel RPM
await axios.post('http://localhost:3000/api/filamentadora', {
  command: 'SET_RPM',
  value: 50  // RPM value 0-100
});
```

```javascript SET_FEED
// Set carriage feed rate
await axios.post('http://localhost:3000/api/filamentadora', {
  command: 'SET_FEED',
  value: 8.5  // Feed rate in mm/s
});
```

</CodeGroup>

## Workflow

1. **Configure Parameters** - Set diameter, length, angle, RPM, feed rate, layers
2. **Preview Pattern** - Visualize winding path in 2D view
3. **Adjust Zoom** - Scale preview for detailed inspection
4. **Start Winding** - Begin automated winding sequence
5. **Monitor Progress** - Track state (running/paused/stopped)
6. **Control Motion** - Manual overrides for rotation and carriage

## API Communication

All commands sent to `http://localhost:3000/api/filamentadora` with JSON payload:

```javascript
{
  command: "COMMAND_NAME",
  value: number  // Optional, for SET_RPM and SET_FEED
}
```

<Warning>
The system uses busy state to prevent command flooding. Only one command can be processed at a time.
</Warning>
